# Wireshark Traffic Analyzer — "What App Are You Really Using?"

**EC 441 Final Project · Spring 2026**

A pcap-driven traffic-fingerprinting tool. It reads a `.pcap` file, groups
packets into flows, computes the network-layer signature of each flow, and
guesses the application that produced it — Zoom, Discord, YouTube, Spotify,
or generic Web browsing — with a transparent, defensible reason for every
guess.

Demo day pitch: *"Even when traffic is encrypted, the shape of the bytes on
the wire still gives the app away. Let me show you."*

---

## TL;DR — run the demo

```bash
cd final_project
bash demo.sh
```

This installs the four Python deps (`scapy`, `matplotlib`, `rich`, `numpy`),
generates synthetic pcaps for all five apps, merges them into a 12-second
"all five apps at once" capture, prints the analysis report to your
terminal, and writes four charts to `reports/mixed/`.

Designed to **work offline** — no classroom Wi-Fi required.

---

## What the demo shows

Running on the included `pcaps/mixed.pcap` (≈8 000 packets, 5.7 MB, five
apps overlapping for 12 seconds) you get:

1. **Transport-layer breakdown** — TCP vs UDP packet and byte counts with
   inline progress bars.
2. **Top server ports** — labelled with the likely service
   (`HTTPS / QUIC`, `Zoom media`, `STUN`, …).
3. **Top destinations** — the IPs each side actually talked to.
4. **Top flows table** — for each flow: protocol, server, packet count,
   total bytes, average size, burstiness, **RTT estimate from the TCP
   handshake**, and the app verdict with confidence percentage.
5. **Per-flow "Why" panels** — every classification comes with the list of
   features that scored, e.g. *"port 8801 in Zoom range 8801-8810 (+4.0)"*.
6. **Four PNG charts** in `reports/mixed/`:
   - `01_tcp_vs_udp.png` — protocol breakdown.
   - `02_packet_sizes.png` — size distribution, log-y, MTU marker.
   - `03_burst_timeline.png` — bytes/100 ms, TCP vs UDP stacked.
   - `04_flow_fingerprints.png` — scatter of avg packet size vs burstiness,
     bubble size = total bytes, colour = app guess. **This is the headline
     image — five apps cleanly separated in one picture.**

## How the classifier works

The classifier is deliberately **transparent**: no ML, no black box. Each
flow is scored against five hand-tuned application fingerprints based on:

| Feature              | What it captures                                          |
|----------------------|-----------------------------------------------------------|
| Transport (TCP/UDP)  | UDP → realtime media; TCP → bulk/web                      |
| Server port          | Zoom 8801-8810, Discord 50000-65535, HTTPS 443, …         |
| Avg / p95 packet size| Tiny ≈ Opus voice; ≈MTU ≈ video chunks                    |
| Burstiness (CV)      | Low CV → steady realtime; high CV → ABR chunked download  |
| Symmetry             | ~1.0 → 2-way call; near 0 → server→client streaming       |
| Total bytes & duration| 1 MB+ over 5 s → video; ≤200 KB short → web resource     |

Each rule contributes a positive (or rarely negative) score with a
human-readable reason; the highest total wins. Confidence is the winner's
share of the total score, and the runner-up is reported alongside.

The fingerprints live in
[`traffic_analyzer/classifier.py`](traffic_analyzer/classifier.py) and are
short enough to read on stage.

### What's the synthetic data?

The pcaps under `pcaps/` are crafted with Scapy so every IP/TCP/UDP
**header** is real and parseable, and the **packet sizes, ports, and
inter-arrival times** match each app's documented pattern (Zoom audio at
~50 Hz on UDP/8801-8810, YouTube ABR chunks on TCP/443, etc.). The
classifier reads them the same way it would read real captures from
`tshark` or Wireshark.

A live demo can also be run on real captures:

```bash
sudo tcpdump -i en0 -w live.pcap   # in another terminal, while using Zoom etc.
python -m traffic_analyzer analyze live.pcap --charts reports/live/
```

## CLI

```bash
# Full demo (generate + analyze + chart):
python -m traffic_analyzer demo

# Analyze any pcap:
python -m traffic_analyzer analyze path/to/file.pcap --charts reports/out/

# Generate one synthetic pcap:
python -m traffic_analyzer generate {zoom|discord|youtube|spotify|web|mixed} out.pcap
```

Run the unit tests with:

```bash
python -m unittest discover tests
```
